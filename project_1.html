Для сравнения характеристик различных ORM были использованы следующие технологии:

Бэкенд

  • Django - платформа, которая позволяет разрабатывать серверные приложения
  • SQLAclhemy - асинхронная программная библиотека на языке Python для работы с реляционными СУБД с применением технологии ORM. Служит для синхронизации объектов Python и записей реляционной базы данны.
  • Tortoise ORM - это простой в использовании asyncio ORM (объектно-реляционный преобразователь), созданный по мотивам Django.
  • SQLite - SQL база данных, которая хорошо подходит для настоящей тестирования настоящей задачи

Процесс разработки

Разработка должна показать производительность различных ORM на базе одной СУБД. Продемонстрировать возможность управлять базами данных с помощью методов объектов в коде и при этом не использовать SQL-запросы :

  • Анализ требований. На этом этапе мы определяли функциональные и технические требования к проекту. Среди прочего мы определили, что в качестве основы для сайта будет использован framework Django. И на его баез будет построена сравнительная таблица
  • Проектирование. На этом этапе мы создали дизайн сайта, это будет одна таблица, в которую буду сведены цифровые характеристики быстродействия основных CRUD функций.
  • Разработка. Подготовили БД состоящую из 50 000 записей, создали модели. При разработке использовали для различных ORM разные методы программирования, как ООп так и функциональные
  • Тестирование. После того как основная часть была готова, мы провели тестирование, чтобы убедиться, что все работает корректно и без ошибок. Это этап включал в себя функциональное тестирование (проверку функциональности проекта), тестирование производительности
